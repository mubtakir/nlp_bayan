# المنطق في لغة البيان (٤): الاستدلال الاحتمالي والتشكيك

<div dir="rtl">

> في هذا الجزء نعرّف **كيف تتعامل لغة البيان مع عدم اليقين** باستخدام الاحتمالات وأدوات التشكيك العربية.
> سنبقى ملتزمين بالعربية قدر الإمكان، مع ملاحظة أن كلمات المنطق الاحتمالي أصبحت الآن **ثنائية اللغة**:
> - سنستخدم الكتلة الهجينة بالشكل العربي `هجين { ... }` في الأمثلة.
> - ويمكن كتابة الحقائق الاحتمالية بأيٍ من `fact[0.8]` أو `حقيقة[0.8]`.
> في الأمثلة أدناه نلتزم بالعربية قدر الإمكان، مع الإشارة إلى الشكل الإنجليزي عند الحاجة فقط.
> بينما يستطيع المترجم فهم المقابل العربي بالكامل في الكود الفعلي.

## ١. الحقائق الاحتمالية `fact[0.8]`

### ١.١ مثال: احتمال نزول المطر

```bayan
هجين {
    # ٩٠٪ احتمال أن يمطر اليوم
    fact[0.9] يمطر("اليوم").

    # ٣٠٪ احتمال أن تهب رياح قوية
    fact[0.3] رياح_قوية("اليوم").
}
```

- الصيغة العامة: `fact[احتمال] علاقة(وسائط...).`
- الاحتمال رقم بين 0 و 1 (مثل 0.9 = 90٪).
- عندما تستخدم هذه الحقائق في قواعد واستعلامات، يقوم محرّك البيان **بتجميع الاحتمالات** على مسار الاستدلال (بالضرب).

### ١.٢ مثال: احتمال حالة مزاجية

```bayan
هجين {
    fact[0.8] نوم_جيد("أحمد").
    fact[0.6] شرب_قهوة("أحمد").

    مزاج_جيد(?شخص) :-
        نوم_جيد(?شخص),
        شرب_قهوة(?شخص).
}
```

- هنا احتمال `مزاج_جيد("أحمد")` يقارب `0.8 × 0.6 = 0.48` (٤٨٪).

## ٢. أدوات التشكيك العربية: ربما، محتمل، غير_محتمل، ممكن، مؤكد

لغة البيان تقدّم خمس أدوات تشكيك عربية مرتبطة بالاحتمال الحالي للحل:

| الأداة | الشرط على p | المعنى التقريبي |
|--------|-------------|------------------|
| `ربما` | `p > 0.5`   | أكثر من ٥٠٪      |
| `محتمل` | `p > 0.7`  | قوي (أكثر من ٧٠٪) |
| `غير_محتمل` | `p < 0.3` | ضعيف جدًا (أقل من ٣٠٪) |
| `ممكن` | `0.2 < p < 0.8` | في الوسط، ليس ضعيفًا جدًا ولا قويًا جدًا |
| `مؤكد` | `p > 0.95` | شبه يقين (أكثر من ٩٥٪) |

هذه الأدوات تُستعمل في الاستعلامات للتحقق من قوة احتمال حدث معيّن.

## ٣. مثال كامل: الطقس غدًا مع أدوات التشكيك

```bayan
هجين {
    # حقائق احتمالية عن الطقس غدًا
    fact[0.75] سيمطر("غدًا").
    fact[0.15] سيتساقط_ثلج("غدًا").

    # استعلامات التشكيك بالعربية
    استعلام ربما("سيمطر", "غدًا").        # نعم (٧٥٪ > ٥٠٪)
    استعلام محتمل("سيمطر", "غدًا").       # نعم (٧٥٪ > ٧٠٪)
    استعلام غير_محتمل("سيتساقط_ثلج", "غدًا"). # نعم (١٥٪ < ٣٠٪)

    # استعلام للتأكد من أن الحدث شبه مؤكد
    استعلام مؤكد("سيمطر", "غدًا").        # لا (٧٥٪ < ٩٥٪)
}
```

- الأسماء (`"سيمطر"`, `"سيتساقط_ثلج"`, `"غدًا"`) بيانات عربية بالكامل.
- أدوات التشكيك (`ربما`, `محتمل`, `غير_محتمل`, `ممكن`, `مؤكد`) عربية.

## ٤. قراءة قيمة الاحتمال رقميًا (للاستخدام المتقدم)

في بعض الأحيان نحتاج الحصول على **قيمة الاحتمال كرقم** داخل القاعدة نفسها، لا فقط كشرط جاهز.

لغة البيان توفر أداة مدمجة `probability(?P)` تعطيك الاحتمال الحالي للحل.

### ٤.١ مثال: مريض في حالة خطرة إذا تجاوز الاحتمال ٧٠٪

```bayan
هجين {
    # حقائق احتمالية عن حالة المريض
    fact[0.8] لديه_إنفلونزا("المريض").
    fact[0.9] لديه_حمى("المريض").

    حالة_خطرة(?شخص) :-
        لديه_إنفلونزا(?شخص),
        لديه_حمى(?شخص),
        probability(?P),
        ?P > 0.7.

    # استعلام: هل حالة المريض خطرة؟
    استعلام حالة_خطرة("المريض")?
}
```

- داخل استنتاج `حالة_خطرة("المريض")` يقوم المحرّك بضرب الاحتمالات (`0.8 × 0.9 = 0.72`).
- `probability(?P)` تربط `?P` بهذه القيمة (حوالي 0.72) ثم نستخدم الشرط العددي `?P > 0.7`.

## ٥. خلاصة هذا الجزء

- عرّفنا **الحقائق الاحتمالية** باستخدام `fact[0.x]` أو المقابل العربي `حقيقة[0.x]` مع علاقات وأسماء عربية.
- استخدمنا أدوات التشكيك العربية `ربما`, `محتمل`, `غير_محتمل`, `ممكن`, `مؤكد` للاستعلام عن قوة الحدث.
- رأينا كيف يمكننا الوصول إلى **قيمة الاحتمال رقميًا** داخل القاعدة باستخدام `probability(?P)` ثم التعامل معه بشروط عددية.

في الأجزاء القادمة سنربط هذا الأساس الاحتمالي مع:
- **الشبكات السببية** (كيف تنتشر هذه الاحتمالات عبر شبكة من الأسباب والنتائج)،
- ومع **الكيانات** و**المعادلات والمشغلات اللغوية**،
- ضمن نموذج أوسع للفكرة–الحدث–النتيجة وطبقات Conceptual LM والتفسير اللغوي.
